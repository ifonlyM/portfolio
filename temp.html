SmartAPC

글로벌 의료진들과 환자의 진료상담을 Zoom 및 다양한 통신매체를 활용하여 중계하는 플랫폼

서비스 유형: 하이브리드 앱

담당 역할: DB 설계, 백엔드 API 개발, 클라우드 서버 구축 및 관리

담당 기간: 2022.11 - 2023.05

개발환경
  • Language: Java8
  • FrameWork: Spring Framework 5.2.3
  • Version Control: git
  • Build tools: maven 4.0.0
  • Deployment tools: File Zila, PuTTY
  • Infrastructure: AWS EC2 (Amazon Linux 2, Apache, Tomcat 8.5), AWS RDS (PostgreSQL), AWS Client VPN

주요 경험

  • Spring MVC에서 제공하는 인터페이스인 HandlerMethodArgumentResolver를 구현하여 사용자 인증 및 메타데이터를 관리하는 반복적인 코드를 줄이고, 코드 재사용성을 확보했습니다. 이 후 작업하는 사내 신규 프로젝트에 모두 이 방법을 적용하여 개발 하였습니다.
  • Enum을 활용한, Entity의 상태 값 관리 기존 사내 프로젝트는 Entity의 상태 값이나, 코드 값을 "01", "02", "03"… 같은 문자열로 비즈니스 로직에 작성 해왔었음. Enum을 활용하여 상태 명, 코드 명으로 비즈니스 로직을 작성할 수 있게끔 프로젝트에 적용함.
    Ex: "05" -≫ eScheduleState.FINISHED.getCode()
  • API 에러 핸들링 프로세스 구축 기존 사내 프로젝트는 서버에러에 대한 대처 프로세스가 없었음. (서버에러 발생시 그냥 404 페이지 보여주고 끝) Enum을 활용하여 에러코드를 정립하고, 컨트롤러 단에서 @ExceptionHandler() 을 활용해 HTTP Status, 에러코드, 에러사유 등 에러 관련 데이터를 반환하여 프론트엔드 단에서 에러 발생에 대처 가능하도록 프로세스 구축
  • AWS EC2, RDS, Client VPN 서비스 사용
  • zoom 마켓에 앱 등록 신청 및 심사 피드백에 대한 모든 대응
